## Assessment

### Spacing

**Issue 1: Arbitrary values, no shared scale.**

The codebase mixes:
- `4px` (player-card inline style line 211)
- `0.65rem / 0.7rem` (waiting-list SCSS)
- `12px / 14px` (player-card)
- `18px / 20px / 22px / 26px / 28px` (multiple)
- `2rem` (main.scss app padding)
- `1.6` (line-height) — not on the 1.5 grid

DESIGN.md prescribes `xs:4 sm:8 md:12 lg:16 xl:24 xxl:32` and `4pt base scale`. The code is **at ~50% adherence**.

Most divergent files:
- `player-card.js` (95-line inline `<style>`): 30+ arbitrary values
- `Lobby.jsx`: 1000+ lines of inline `style={{}}` — every value is a one-off
- `seat-modal.jsx`: 8 inline style blocks with one-off values

**Issue 2: Equal padding everywhere = no rhythm.**

The `waiting-list-item` uses `padding: 0.65rem 0.7rem` — uniform 10.4px / 11.2px (not on the 4pt scale). The grid `gap: 0.65rem` between rows is the same value as the row's padding, which makes rows feel glued to each other rather than grouped.

The `side-panel` template uses `<section class="panel-section">` but no `gap` between sections — they sit on top of each other.

**Issue 3: Related elements sometimes have generous separation, sometimes tight.**

`panel-section` children have no margin between them, but each `panel-section__label` has `margin:0` forced inline (line 81 of `side-panel.js`) — this fights a CSS rule that probably already exists. Investigate.

### Visual hierarchy

**Issue 4: Hierarchy relies on color and weight, not space.**

The side-panel:
- "Panel de Mesa" header (h3-ish) and the radio group are separated by `<section>` boundary but no margin.
- "Sesión" section follows with no space.
- "Lista de Espera" section follows with no space.

Result: it reads as a wall of glass, not a hierarchy of zones. **The squint test fails** — primary, secondary, groupings aren't identifiable.

**Issue 5: Numbers in player-card (Stack, Compras, Tiempo) are visually equal weight.**

All 4 stats in player-card are the same `font-size: 17px` and same `font-weight: 700`. The "Stack" stat (operator's #1 question) is not visually prioritized. Money (`#d4af37`) is the only differentiator, and that's by color, not by size/space.

**Recommendation (queued for polish):** Stack stat should be 24-28px, others 17px. Money color stays. Spacing between stats should drop from `1px` (the grid gap) to `0` (or use a divider instead of bg).

### Grid & structure

**Issue 6: No explicit grid on operator pages.**

`src/pages/tables-manager.js` mounts the side-panel + poker-table. The layout is "centered, max-width 1400px" (from `_base.scss`). The side-panel floats absolute (left), poker-table centered. On viewports >1400px, side-panel is 280px wide, table 800px, leaving 320px of dead space on the right. **Asymmetric and unintentional.**

**Issue 7: Glass card "grid" inside player-card is 2×2 stats with no semantic structure.**

`.pc-modal-stats` is `display: grid; grid-template-columns: 1fr 1fr; gap: 1px`. The 1px gap is achieved by background-color bleeding through, with `1px solid rgba(255,255,255,0.06)` borders. Visually busy. Should be a single divider between rows, or a clean border-bottom on each stat.

### Rhythm & variety

**Issue 8: Every row in the waiting list is the same height, same padding.**

The waiting list's `gap: 0.65rem` is the same as the row's padding, so they look like a continuous strip. Should be `gap: 8px` (or 12px) with `padding: 12px 14px` per row.

**Issue 9: `side-panel` sections have no rhythm.**

Three `.panel-section` blocks stacked with no margin. The whole panel is "one big thing". The session pill, the timer, the waiting list, the status radio — they should be **discrete zones** with `margin-top: 20-24px` between them.

### Density

**Issue 10: Right balance overall, but Lobby.jsx is too sparse.**

`Lobby.jsx` (1602 lines) has lots of vertical breathing room — `padding: 40px` on the table, `margin: 2rem` between sections, generous `gap: 24px` on grids. The lobby is a public surface; some air is appropriate. But the operator (gestor) is dense, and the lobby sometimes inherits the same breathing room by mistake.

**Issue 11: `table-form` (table settings modal) is over-spaced.**

Long labels with `margin-bottom: 1rem` (16px) is fine, but the input groups are 20px+ apart, making the modal scroll when 6 fields would fit in 600px.

## Layout Improvement Plan

### Spacing system

Adopt `DESIGN.md` spacing scale as the canonical:
- `xs: 4px` (inline separators)
- `sm: 8px` (input internal padding, tight sibling gap)
- `md: 12px` (default control padding, list item padding)
- `lg: 16px` (card padding, section gap minimum)
- `xl: 24px` (between major zones, modal padding)
- `xxl: 32px` (page-level, hero spacing)

No more arbitrary values. **All inline styles in `player-card.js`, `Lobby.jsx`, `seat-modal.jsx`, and `side-panel.js` get migrated to classes that use this scale.**

### Hierarchy strategy

| Surface | Primary | Secondary | Tertiary |
|---|---|---|---|
| Gestor / Side panel | Mesa activa badge + timer | Status radio + waiting count | Settings/Export icons |
| Player card | Stack (gold) + Avatar | Compras / Tiempo / Ingreso (white) | Action buttons (ghost) |
| Seat modal | Confirm action | Form fields | Cancel |
| Waiting list | Sit button (gold/green hover) | Player name | Remove (red hover) |

**Hierarchy is achieved by: size + weight + spacing, not by color alone.** Money in gold is the only color priority.

### Layout approach

- **Operator root**: CSS Grid with named areas: `side-panel | main | side-context` (currently right column is empty/missing).
- **Side panel internal**: Flex column with `gap: xl` (24px) between sections.
- **Player card**: Stacked sections, no nested grid in the stats area.
- **Lobby**: Bento layout, breaking the 1-column flow at `lg` breakpoint.

### Rhythm

- **Tight** (4-8px): between sibling fields in a form, between rows in a list.
- **Default** (12-16px): between related actions, between stats in a card.
- **Generous** (24-32px): between sections in a panel, between unrelated cards.

## Specific Layout Changes Queued for Polish

1. **side-panel.js template → SCSS**: Move inline styles to `.wl-header`, `.wl-header__actions`; add `gap: var(--space-md)` between `.panel-section`s.
2. **player-card.js inline `<style>` → player-card.scss**: Migrate 95 lines; align with the rest of the design system.
3. **waiting-list.scss**: Rename selectors to match rendered HTML (`wl-item-new` etc.) OR rename HTML to match SCSS (`waiting-list-item` etc.). Pick one.
4. **SeatModal.jsx**: Extract 8 inline `style` blocks to `seat-modal.scss`.
5. **Lobby.jsx**: 1000+ lines of inline styles — this is too large for a single polish pass. Document as **deferred** and add `// TODO: migrate to scss` markers.
6. **player-card stats grid**: Increase Stack stat to 24px; remove the 1px gap that creates busy background bleed; use a single divider between rows.
7. **side-panel section spacing**: Add `margin-top: 20px` between `.panel-section`s (or convert to flex column with gap).
8. **Add `:focus-visible` ring** to all player-card buttons (uses `--focus-ring-gold` from DESIGN.md).
9. **Cache `queryRefsFrom` lookups** in `startAbsenceCounter` (3 queries per second → 0).
